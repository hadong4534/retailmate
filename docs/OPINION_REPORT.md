## 1. 현재 상태 한눈에

- **스택**: Next.js 16 + React 19 + Supabase, 모바일 우선 PWA. 약 24,500줄.
- **배포**: 모든 변경은 `refactor` 브랜치 → Vercel preview. **운영(retailmate.io=main)은 미반영** (최종 확정 후 1회 merge 예정).
- **DB**: 라이브 Supabase(21테이블). 추가·비파괴 마이그레이션만 반영(NDA/cancelled, 함수 search_path 하드닝).
- **검증 방식**: 모든 코드 변경은 `tsc` + 프로덕션 빌드 통과로 게이트. 런타임(로그인·DB쓰기·모바일 실기)은 preview 테스트 권장.

## 4. 추가하면 좋은 기능 (신규 제안)

1. **상품/메뉴 단위 매출**(products·sale_items 테이블 이미 존재, 미사용) — 메뉴별 매출·원가·마진 분석. 대시보드 가치 급상승.
2. **AI 주간/월간 리포트 자동 생성** — 매출·비용·인건비를 AI가 요약해 카카오/이메일로 발송(스케줄). 차별화 포인트.
3. **직원 근무 스케줄(work_schedules 테이블 이미 존재, 미사용)** — 주간 시프트 편성 + 예상 인건비 미리보기.
4. **영수증 사진 OCR**(지출 첨부 placeholder 실구현) — 사진→금액·항목 자동 인식.
5. **알림/푸시**(user_notification_prefs 이미 존재) — 마감 미입력·계약 만료·재고 알림 등 PWA 푸시.
6. **다점포 통합 대시보드** — 여러 매장 합산 리포트(멀티스토어 이미 지원).
7. **세무/신고 도우미** — 부가세·종합소득세 기간 자동 집계 + 자료 내보내기.

## 5. 리빌딩 추천 (현재 있지만 다시 설계 권장)

1. **급여 모듈** — 현재 메모리 단순계산 + 미완성. `payrolls` 기반으로 정식 재설계(법정수당·세액·명세서).
2. **AI 이미지 파이프라인** — fire-and-forget 구조를 큐 기반으로 재설계.
3. **권한/데이터 접근** — service-role 남용 → RLS 신뢰 구조로 정리(보안·유지보수성).
4. **계약 모듈 데이터 모델** — 근로계약과 NDA가 한 테이블 혼재. 공통/전용 필드 분리 또는 뷰 정리 검토.
5. **검증·시간·포매터 공통화** — zod 단일 소스, KST 유틸 단일화, 전화/사업자번호 포매터 통합(중복 제거).

## 7. 결론 / 추천 진행 순서

디자인·핵심 버그(P0)는 정리됐습니다. 다음은 **(1) 급여 엔진 완성 → (2) service-role 정리 → (3) AI 이미지 큐 → (4) 보안 advisor** 순으로, 각 단계 preview 실기 테스트를 거쳐 진행하길 권합니다. 그 뒤 **신규 기능 중 "상품 단위 매출"과 "AI 자동 리포트"** 를 추가하면 제품 경쟁력이 크게 올라갑니다.

운영 반영은 위 체크리스트 통과 후 한 번에 진행하면 됩니다.
