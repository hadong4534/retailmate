# RetailMate 입력 필드 전수조사 & UX 개선 기획서

> 작성일 2026-05-31 · 복구 소스 정독 기반 · 목적: "빈칸(입력) 데이터" 전수 파악 + 효율적 입력/수정/삭제 재설계

## 0. 한눈 요약 — 핵심 문제

사장님이 지적한 "매출 데이터를 하나하나 고르고 쓰고, 수정·삭제가 제대로 안 된다"는 체감은 코드로 확인됩니다. 공통 문제:

1. **매출 입력이 6개 결제수단을 매번 다 펼침** — 현금만 받는 가게도 5개 빈칸을 매번 스크롤.
2. **첫 필드 autofocus 없음** — 거의 모든 폼이 시작에 탭 1번 낭비 (예외: 직원 시급 인라인 편집).
3. **빠른 금액 버튼 없음** — 모든 금액을 숫자 키패드로 한 자리씩 타이핑.
4. **매출 수정 = 전체 삭제 후 재삽입** — 한 칸 고치려고 그날 전체 row 삭제·재생성(감사이력 소실, 중간 실패 시 그날 매출 증발). 별도 풀페이지 이동 필요, 인라인 편집 없음.
5. **삭제 UX 제각각** — 매출은 인라인 2단계 확인, 비용·직원은 브라우저 `confirm()`/`alert()`. 되돌리기(undo)·토스트 없음.
6. **수정 경로 자체가 없는 화면** — 비용·공지·계약은 작성 후 수정 불가(삭제 후 재입력해야 함).
7. **소프트/하드 삭제 혼재** — 매출·비용은 하드 삭제, 직원 퇴사는 소프트 삭제인데 안내문은 "완전 삭제"로 잘못 표기.

가장 잘 만들어진 인라인 편집 패턴은 `employees/WageEditor.tsx`(연필→인라인 input→autofocus→저장/취소). **이 패턴을 매출/비용에 이식하는 것이 가장 큰 레버.**

## 1. 화면별 입력 필드 인벤토리

### 1-1. 매출 입력 (신규) — `sales/new/SaleNewForm.tsx` → `createSales`
데이터 모델: **하루·결제수단당 sales row 1개** (6채널 = 최대 6 row, 같은 sale_date/memo 공유). 상품 단위 입력 없음("추후 지원").

| 필드 | 타입 | DB 컬럼 | 필수 | 기본값 | 검증 |
|---|---|---|---|---|---|
| 날짜 | date | sales.sale_date | (기본) | 오늘(KST), max=오늘 | max만 |
| 카드/현금/현금영수증/계좌이체/배달앱/기타 | money ×6 | sales.channel + amount | 채널별 | 0 | >0이면 저장 |
| 메모(선택) | textarea | sales.memo (모든 row 복제) | no | '' | max 200 |
| 합계 | 계산(읽기) | — | — | 합 | 합=0이면 저장 비활성 |

문제: 6채널 상시 노출, autofocus 없음, 빠른금액 없음, "내가 쓰는 결제수단" 기억 없음, 다중일 입력 없음, "어제처럼" 프리필 없음, 메모가 row마다 중복 저장.

### 1-2. 매출 수정 — `sales/edit/[date]/EditDayForm.tsx` → `updateDailySales`
- 같은 6필드 풀페이지 재렌더 → 저장 시 **그날 store+date 전체 delete 후 non-zero 채널 재insert** (service-role).
- 문제: 리스트 인라인 편집 불가 / 파괴적 재작성(감사이력·row identity 소실, 중간 실패 위험) / 날짜 변경 불가 / 단일 채널만 삭제 불가.

### 1-3. 매출 삭제 — `deleteDailySales`
- 수정 페이지 안에서만, 2단계 인라인 확인 → 그날 전체 하드 삭제. 단일 채널 삭제 불가, undo 없음, 리스트에서 바로 삭제 불가.

### 1-4. 비용 입력 — `expenses/new/ExpenseNewForm.tsx`
| 필드 | 타입 | DB | 필수 | 기본 |
|---|---|---|---|---|
| 날짜 | date | expense_date | yes | 오늘 |
| 카테고리 | 8버튼 그리드 | category | yes | material |
| 항목명 | text | item_name | no | '' (max50) |
| 금액 | money(lg) | amount | **yes(>0)** | 0 |
| 결제수단 | 4버튼 | payment_method | no | 카드 |
| 거래처/메모 | text/textarea | vendor/memo | no | '' |
| 영수증 첨부 | (비활성 placeholder) | — | — | "곧 지원" |
- 카테고리/결제 타일 UX는 좋음. autofocus·빠른금액 없음. **수정 경로 없음**(삭제 후 재입력). 삭제는 `confirm()` 하드 삭제(단일 row는 됨), undo 없음. 영수증 첨부 죽은 placeholder.

### 1-5. 근로계약서 마법사 — `contracts/new/Wizard.tsx` (4단계)
기본정보(이름·휴대폰·계약형태·시작/종료일·근무장소·담당업무) / 근무시간(요일칩·시작종료·휴게) / 임금(형태·금액·주휴·4대보험·지급일·지급방법·연차·부가) / 미리보기+사장서명. 단계별 검증·타입별 기본값 우수. **수정 경로 없음**(재발행). 지급일 number 입력의 "빈칸=0 후 blur clamp" 어색 → settings처럼 select 권장. 1단계 휴대폰 자릿수 미검증(NDA는 검증 — 불일치).

### 1-6. NDA 마법사 — `contracts/new/nda/NDAWizard.tsx` (2단계)
이름·휴대폰(≥10자리)·시행일·유지기간(1~10년, 기본3)·비밀정보범위 + 사장서명. 링크 7일 만료(근로계약은 무기한 — 정책 불일치). 수정 없음. ※ 서명 URL 생성 버그 있음(별도 보고서 참조).

### 1-7. 계약 서명(직원) — `contracts/[id]/sign/SignFlow.tsx`
auth(카카오/이메일+SMS인증) → review(약관3·서명) → done. 모바일 키보드 힌트·autocomplete 우수. SMS 재전송 쿨다운 없음(가입 화면엔 60초 — 불일치).

### 1-8. 공지 작성 — `notices/new/NoticeForm.tsx`
제목(max100)·내용(max2000)·대상(전체/직원만)·만료일(선택)·상단고정. 깔끔. **수정 경로 없음**.

### 1-9. 설정 — `settings/SettingsClient.tsx` (5탭)
매장정보(매장명·사업자번호[마스킹]·업종·주소[자동 lat/lng]·반경 슬라이더·영업시간·월목표) / 계정(아바타·이름·이메일[비활성]·휴대폰) / 알림(토글4, 토글마다 즉시 저장—과다 호출) / 급여세금(select4) / 보안(비번변경). 앱에서 가장 잘 만든 폼. "최근 저장 정보"가 저장 전에도 "모두 저장됨" 표기(오해 소지).

### 1-10. 매장 생성 — `stores/new/NewStoreForm.tsx`
매장명·업종·사업자번호(여기선 마스킹 없음 — settings와 불일치)·주소·상세주소. monthly_target=0 하드코딩.

### 1-11. 로그인/가입 — `(auth)/login`, `(auth)/signup`
로그인: 이메일·비번·자동로그인·카카오. 가입: 이메일·비번(≥8)·이름·휴대폰+SMS(60초 쿨다운)·인증번호·약관2. autocomplete 우수. 가입 화면 `formatPhone`이 PhoneInput과 중복.

### 1-12. 직원 — `WageEditor.tsx`(인라인 편집 모범) / `MemberActions.tsx`(⋯ 편집·승격·퇴사)

## 2. 개선 기획 (우선순위)

### A. 즉시 개선 (저비용·고효과)
1. **매출 리스트 인라인 편집 + 채널별 삭제** — WageEditor 패턴 이식. 표에서 숫자 탭→그 자리서 수정. 풀페이지 이동·파괴적 재작성 제거.
2. **모든 MoneyInput에 빠른금액 버튼**(+1천/+1만/+10만/지우기). "타이핑 귀찮음"의 최대 해결책.
3. **신규 입력 폼 첫 필드 autofocus**.
4. **안 쓰는 매출 채널 접기** — "내가 쓰는 결제수단"을 매장 설정에 1회 저장 → 해당 채널만 노출, 나머지는 "+ 다른 결제수단". 현금가게는 1칸만 보임.
5. **`confirm()`/`alert()` → 통일된 인앱 확인 + 토스트(undo)**. 삭제는 소프트 삭제+undo로 전환.
6. **비용 수정 폼 추가**(ExpenseNewForm 재사용).
7. **오해 문구 수정**(설정 "저장됨" 선표기, 퇴사 "되돌릴 수 없음").

### B. 본격 개편
8. **매출 입력을 "오늘 매출" 단일 시트로**: 최근일 채널 구성 프리필("어제처럼"), 날짜 칩(어제/오늘), 채널별 빠른금액, 실시간 합계, 1회 저장. 목표: 평상시 하루 입력 <10탭.
9. **다중일(주간 그리드) 입력** — 행=날짜, 열=채널. 며칠 밀린 입력을 한 번에.
10. **비파괴 매출 수정** — (store,date,channel) upsert로 전환, created_by/timestamp 보존, 중간 실패에도 그날 매출 안전.
11. **메모 정규화** — 채널 row마다 중복 저장하던 메모를 하루 1개로 분리(별도 컬럼/테이블).
12. **공유 폼·검증 레이어 표준화**(zod 클라+서버 공유 — 1단계 착수 완료, 확대), 전화/사업자번호 포매터 단일화, 공통 확인/토스트 컴포넌트.
13. **약속만 하고 비활성인 입력 출시 또는 제거**: 비용 영수증 사진 업로드, 상품 단위 매출.

### 데이터 모델 권고
- 매출 메모를 채널 row에서 분리(하루 단위) → 수정 단순화·중복 제거.
- (store_id, sale_date, channel) 유니크 + upsert로 비파괴 수정.
- 소프트 삭제 컬럼(deleted_at) 도입 → undo·휴지통 가능.
- 미사용 테이블 `products/sale_items` 활용 시 상품 단위 매출 확장 경로 확보.
