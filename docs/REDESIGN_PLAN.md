# RetailMate 리디자인 기획서 (PC + 모바일)

> 목적: "요즘식 앱"으로의 대대적 UI 개편. 사장님·직원 모두 쉽게, 모든 기능이 잘 작동. 모바일 AI 접근 복구. Higgsfield/이미지 생성 활용 계획 포함.
> 동봉: `mockups/redesign_concepts.html` (브라우저로 열어 3개 시안 × PC/모바일 확인)

## 1. 현재 문제 (UI 관점)
- **모바일에서 AI 사용 불가**: `AppShell`의 하단 탭(BOTTOM_NAV 5개)에 AI 항목이 없고, AI는 PC 사이드바 그룹에만 존재. 모바일 사용자는 챗/포스터/브랜드/드라이브 진입로가 사실상 없음(프로필 드롭다운 일부만). → **치명적**.
- 정보 밀도가 낮고 카드가 제각각(디자인시스템 §6: 미마이그레이션 카드 30+).
- 매출/비용 입력이 폼 위주(위 입력 감사서 참조).
- 데스크탑은 다크네이비 사이드바 + 라이트 본문, 모바일은 상단바+하단탭으로 일관성 부족.
- AI 모델명("Claude Sonnet 4.6") UI 노출 등 군더더기.

## 2. 디자인 원칙 (요즘식)
1. **모바일 퍼스트, 엄지 존(thumb-zone) 우선** — 핵심 액션을 하단에.
2. **단일 강조색 + 뉴트럴** — 기존 브랜드 블루(#2563EB) 유지하되 면/그림자 정리, 라운드 16~20px, 큰 터치 타깃.
3. **AI를 1급 시민으로** — 모바일 하단 탭 중앙에 AI(스파클) FAB, 어디서나 1탭.
4. **"입력은 빠르게, 보기는 풍부하게"** — 빠른입력 시트 + 인사이트 카드.
5. **접근성** — 16px+ 폰트, 명도 대비 AA, prefers-reduced-motion 존중.

## 3. 리디자인 시안 3종 (동봉 HTML에서 시각 확인)

### 시안 A — "클린 SaaS" (정석/안전)
- 라이트 배경 + 카드, 좌측 아이콘 레일(PC). 모바일 하단탭 5 + 중앙 AI FAB.
- 톤: 차분한 블루/슬레이트. 위험 적고 빠르게 적용 가능. 기존 자산 재활용 최대.

### 시안 B — "다크 대시보드" (프리미엄/데이터 중심)
- 다크 네이비 베이스 + 네온 블루/시안 포인트, 글래스 카드. 큰 숫자 KPI, 미니 차트 풍부.
- 톤: 힙하고 데이터 강조. 야간 매장/세련된 인상. AI 오브(orb)와 잘 어울림.

### 시안 C — "토스/뱅킹 스타일" (초간결·모바일 극대화)
- 흰 배경, 굵은 타이포, 한 화면 한 동작. 큰 버튼·리스트, 스와이프 액션(수정/삭제).
- 톤: 비개발자 사장님·직원이 가장 직관적. 입력 흐름 최소 탭에 최적.

> 권장: **모바일=C 지향, PC=A/B 혼합**. 최종은 사장님이 시안 보고 택1 → 그 방향으로 컴포넌트 토큰 확정.

## 4. 모바일 AI 복구 (즉시 필요)
- `AppShell` BOTTOM_NAV를 [홈, 매출, **AI(중앙 FAB)**, 직원, 더보기]로 재구성. 중앙에 스파클 FAB(그라데이션) → `/ai` 허브.
- `/ai` 허브를 모바일 풀스크린으로: 챗·포스터·브랜드·드라이브 카드.
- 챗 화면 입력창이 하단탭에 가리던 문제(P0)는 이미 부분수정 — 새 FAB 구조로 근본 해결.

## 5. 핵심 화면 재설계 포인트
- **홈(대시보드)**: 오늘/이달 매출·이익 큰 숫자 + 한 줄 AI 인사이트 + "오늘 매출 입력" 1탭 CTA + 빠른 액션 그리드.
- **매출**: "오늘 매출" 빠른 시트(어제처럼 프리필, 빠른금액, 실시간 합계), 일별 리스트는 인라인 편집/스와이프 삭제.
- **AI**: 모바일 풀스크린 챗 + 추천 질문 칩, 포스터 생성 진행상태 토스트.

## 6. Higgsfield / 이미지·영상 생성 활용 계획
> 앱 UI 목업 자체는 HTML/CSS로 만드는 것이 정확하고 수정가능합니다(AI 이미지로 만든 가짜 스크린샷은 구현에 못 씀). 따라서 **AI 생성은 "이미지·모션이 실제로 필요한 자산"에 집중**합니다.

권장 사용처(방향 확정 후 실행):
1. **랜딩/마케팅 히어로 이미지** — gpt 이미지 모델로 매장 사장님·모던 대시보드 무드 비주얼.
2. **온보딩/빈 상태(empty state) 일러스트** — 일관된 스타일 세트.
3. **앱 소개 모션(영상/GIF)** — Higgsfield로 기능 데모 루프(매출 입력→인사이트), 스토어 등록 영상.
4. **PWA 스플래시/아이콘 모션**.

실행 절차(사용자 승인 후): 방향(A/B/C) 확정 → 무드보드 프롬프트 확정 → gpt 이미지로 정지 컷 N장 → 선택 컷 기반 Higgsfield 모션. 크레딧이 들기 때문에 **방향 확정 전에는 생성하지 않음**(이번 자율 작업에서 보류).

## 7. 적용 로드맵
- (1) 모바일 AI 복구 + 하단탭 재구성 — 가장 시급, 작음.
- (2) 디자인 토큰/컴포넌트 정리(`@/components/app`로 수렴) — 시안 확정 후.
- (3) 매출 빠른입력 시트 + 인라인 편집 — 입력 감사서 A/B 항목과 연계.
- (4) 화면별 점진 적용 + preview 배포로 모바일/PC 실기 검증 반복.
